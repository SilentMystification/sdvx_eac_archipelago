# SDVX EAC Archipelago

A DLL mod for **Sound Voltex EXCEED GEAR (sv6c)** that connects the game to an [Archipelago](https://archipelago.gg) multiworld session. Buttons and knobs start locked and are unlocked as Archipelago items are received. Song clears send location checks to the server.

---

## Requirements

- **Visual Studio 2022 Build Tools** (or full VS 2022) with the **Desktop development with C++** workload
  - Download: https://aka.ms/vs/17/release/vs_buildtools.exe
- Sound Voltex EXCEED GEAR (sv6c) with bemanitools / dinputhook-sdvx

---

## Building

`build.bat` auto-detects your Visual Studio installation via `vswhere.exe` — no path editing needed.

```bat
build.bat
```

Output is staged into `deploy\`. See the **Deployment** section below.

## Deployment

Running `build.bat` populates the `deploy\` folder with everything needed:

```
deploy\
  version.dll                  ← game mod
  sdvx_ap_debug.exe            ← debug UI
  archipelago.ini              ← edit this before launching the game
  generate-apworld.bat         ← AP world generator
  tools\
    extdrm.exe                 ← place here manually (not committed to git)
    generate_song_data.py
  ap_world\
    __init__.py
    sdvx_template.yaml
    data.py                    ← generated by generate-apworld.bat
```

### Game mod

Drag the entire contents of `deploy\` into the game's `modules\` folder, then edit `archipelago.ini` and set `slot = YourName` to match your slot in the AP room.

### AP world

#### 1. Generate the AP world data

Run `deploy\generate-apworld.bat` once after building, and again after each game update.

Open the script and update the game path at the top if needed:

```bat
set "GAME_DIR=C:\Games\SOUND VOLTEX EXCEED GEAR"
```

The script will:
1. Extract `music_db.xml` from the game data using `deploy\tools\extdrm.exe`
2. Parse it and write `deploy\ap_world\data.py`

#### 2. Install to Archipelago

Copy the `deploy\ap_world\` folder into your Archipelago `lib\worlds\` directory and name it `sdvx`:

```
C:\ProgramData\Archipelago\lib\worlds\sdvx\
```

## Configuration

Edit `archipelago.ini` (included in `deploy\`) before launching the game:

```ini
[Archipelago]
host     = archipelago.gg   ; server hostname or IP
port     = 38281
slot     = YourName          ; must match your slot name in the AP room
password =                   ; leave blank if the room has no password

[Game]
goal_clears = 30             ; number of song clears needed to finish

[Debug]
enabled  = 0                 ; set to 1 to write sdvx_ap.log next to the DLL
```

## How it works

- The mod loads as a `version.dll` proxy, forwarding all real version API calls to the system DLL.
- It hooks `sdvx_io_get_input_gpio` and `sdvx_io_get_spinner_pos` in `sdvxio.dll` to gate inputs until unlocked by AP items.
- It hooks `property_destroy` in `avs2-core.dll` to intercept `game.sv6_save_m` e-amusement packets and detect song clears.
- The AP client connects over WebSocket and handles item/location sync in a background thread.
